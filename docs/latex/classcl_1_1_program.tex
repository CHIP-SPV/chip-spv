Construct a program object from a list of devices and a per-\/device list of binaries. 
\begin{DoxyParams}{Parameters}
{\em context} & \mbox{\hyperlink{class_a}{A}} valid Open\+CL context in which to construct the program. \\
\hline
{\em devices} & \mbox{\hyperlink{class_a}{A}} vector of Open\+CL device objects for which the program will be created. \\
\hline
{\em binaries} & \mbox{\hyperlink{class_a}{A}} vector of pairs of a pointer to a binary object and its length. \\
\hline
{\em binary\+Status} & An optional vector that on completion will be resized to match the size of binaries and filled with values to specify if each binary was successfully loaded. Set to CL\+\_\+\+SUCCESS if the binary was successfully loaded. Set to CL\+\_\+\+INVALID\+\_\+\+VALUE if the length is 0 or the binary pointer is NULL. Set to CL\+\_\+\+INVALID\+\_\+\+BINARY if the binary provided is not valid for the matching device. \\
\hline
{\em err} & if non-\/\+NULL will be set to CL\+\_\+\+SUCCESS on successful operation or one of the following errors\+: CL\+\_\+\+INVALID\+\_\+\+CONTEXT if context is not a valid context. CL\+\_\+\+INVALID\+\_\+\+VALUE if the length of devices is zero; or if the length of binaries does not match the length of devices; or if any entry in binaries is NULL or has length 0. CL\+\_\+\+INVALID\+\_\+\+DEVICE if Open\+CL devices listed in devices are not in the list of devices associated with context. CL\+\_\+\+INVALID\+\_\+\+BINARY if an invalid program binary was encountered for any device. binary\+Status will return specific status for each device. CL\+\_\+\+OUT\+\_\+\+OF\+\_\+\+HOST\+\_\+\+MEMORY if there is a failure to allocate resources required by the Open\+CL implementation on the host. \\
\hline
